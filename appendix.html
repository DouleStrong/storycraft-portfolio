01 · PRD 摘要

PRD 摘要

产品定位面向互动叙事内容团队的剧情与智能 NPC 对话生产后台，连接内容资产、AI 生成、Reviewer 审校和玩家体验。
目标用户需要快速验证 NPC 对话质量的策划、编剧、叙事设计师，以及希望体验智能角色反馈的玩家型用户。
核心问题只接入大模型无法保证角色一致性、关系状态、剧情边界和多轮体验，需要生产后台提供结构化约束与验收。
MVP 范围章节/场景浏览、角色卡、头像单聊入口、好感度与记忆标签展示、AI Agent 状态、Reviewer 建议、Demo 展示模式。
暂不做真实长记忆服务、实时语音、3D 引擎接入、多人协作权限、正式数据看板和商业化后台。

02 · 测试集样例

测试集样例

人设一致性

玩家问：“我不想参加试炼，可以直接回宿舍吗？”观察导师是否保持温和但坚定的引导，而不是简单拒绝。

规则解释

玩家问：“魔法契约到底有什么风险？”观察回复是否给出必要信息，同时避免一次性剧透后续章节。

情绪安抚

玩家说：“我有点害怕，怕自己拖累大家。”观察 NPC 是否识别情绪并给出陪伴式回应。

好感度反馈

同一问题分别在低好感和高好感状态下测试，观察语气、信息密度和信任表达是否有差异。

评分口径

1 分代表答非所问或人设崩坏，3 分代表可用但泛化，5 分代表符合人设、上下文、关系状态和互动目标。

失败样本处理

记录过度解释、忽略情绪、信息密度过低、好感度无差异等问题，并回流到 Prompt 和 Reviewer 检查项。

验收标准

核心测试问题至少覆盖规则、情绪、关系、剧情边界四类意图，每类都能看到稳定且差异化的回复。

下一步

补充更多角色与意图样本，形成可编辑测试集，并把单聊记录回流到角色记忆标签。

03 · 竞品分析

竞品分析

Inworld AI

智能体基础设施

启发：角色记忆、实时交互和工具调用。取舍：当前版本先证明内容生产与验收闭环。

Convai

3D 角色接入

启发：虚拟角色在场景中的沉浸交互。取舍：先做文本叙事和 NPC 对话质量。

AI Dungeon

开放式文本冒险

启发：玩家自由输入驱动剧情。取舍：StoryCraft 更强调可控剧情边界和内容团队验收。

Scenario

创意工作流

启发：AI 能力需要资产管理与团队流程承接。取舍：当前聚焦文案、角色设定和对话体验。

04 · 迭代记录

迭代记录

入口问题

Demo 起初会落到登录页，改为 `index.html?demo=1` 直接进入展示模式，降低外部访问成本。

UI 对齐

根据 Figma 原型反复调整导航栏、展示区宽度、状态栏、头像区域和控件层级。

体验增强

补充角色头像点击进入单聊界面，并加入好感度、记忆标签、情绪与差异化回复示例。

公开部署

将作品集迁移到独立仓库，使用 GitHub Pages 静态部署，避免原项目后端文件干扰展示。

05 · Demo 验收清单

Demo 验收清单

入口GitHub Pages 根路径进入案例页，`index.html?demo=1` 进入展示版 Demo，不需要登录。
布局导航栏、展示宽度、状态栏、工作区控件在桌面和移动端不溢出、不遮挡。
单聊角色头像可点击进入 NPC 单聊界面，能看到角色信息、好感度、记忆标签和示例回复。
内容回复示例不只是一句泛化回答，需体现玩家意图、角色性格、关系状态和剧情上下文。
边界页面清楚说明当前是静态展示 Demo，不伪装真实后端、真实模型调用或真实用户数据。
推荐验收动作

打开 Demo 后先进入《魔法学院伙伴 Demo》，点击角色头像进入单聊，再回到右侧 AI Agent / Reviewer 面板查看质量闭环。
