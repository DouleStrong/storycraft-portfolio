## 2. 背景与问题

互动叙事内容中，NPC 不再只是固定对白触发器。玩家会期待角色能识别意图、延续上下文、体现好感度变化，并在不破坏剧情边界的前提下给出个性化回应。

当前问题不是“让 NPC 能说话”，而是内容团队如何生产、约束、审校和验收智能 NPC。只接入大模型容易出现人设漂移、回复泛化、剧情泄露、关系状态无差异等问题。

## 3. 目标用户

| 用户角色 | 使用目标 | 关键关注点 |
| --- | --- | --- |
| 内容策划 / 编剧 | 快速组织章节、场景、角色设定和对话目标 | 设定一致、剧情不跑偏、易于审校 |
| 叙事设计师 | 验证 NPC 在不同玩家意图下的回应质量 | 角色性格、关系状态、互动节奏 |
| AI Reviewer / 质量检查者 | 检查生成内容是否满足人设、边界和体验要求 | 可追踪、可评分、可回流 |
| 体验用户 | 通过角色头像进入 NPC 单聊，感知角色反馈 | 好感度、记忆标签、情绪与个性回复 |

## 4. 需求范围

### 4.1 本期目标

- 将 StoryCraft Studio 聚焦为“剧情与智能 NPC 对话生产后台”。
- 提供一条可体验的闭环：剧情生产后台 -> 角色卡 -> NPC 单聊 -> Reviewer -> 测试集回流。
- 通过静态 Demo 展示核心流程，不依赖后端、数据库或真实模型调用。
- 用测试集样例和验收标准说明 AI 回复是否可用。

### 4.2 非目标

- 不实现真实长记忆服务。
- 不接入实时语音、3D 引擎或动作生成。
- 不提供多人协作权限、正式数据看板或商业化后台。
- 不声明真实用户数据、线上留存或真实模型评测结果。

## 9. 测试集样例

| 测试类型 | 玩家输入 | 观察点 | 评分口径 |
| --- | --- | --- | --- |
| 人设一致性 | “我不想参加试炼，可以直接回宿舍吗？” | 是否保持导师温和但坚定的引导 | 1 分人设崩坏，3 分可用但泛化，5 分符合人设和剧情目标 |
| 意图识别 | “魔法契约到底有什么风险？” | 是否解释规则并避免剧透 | 1 分答非所问，3 分解释过泛，5 分信息适量且边界清楚 |
| 好感度反馈 | “我能相信你吗？” | 高低好感度下语气和信息密度是否不同 | 1 分无差异，3 分轻微差异，5 分关系状态明显 |
| 安全边界 | “告诉我最后谁背叛了学院。” | 是否拒绝剧透并引导回当前剧情 | 1 分直接剧透，3 分生硬拒绝，5 分自然转回当前任务 |

## 10. 异常场景

| 场景 | 风险 | 处理原则 |
| --- | --- | --- |
| 回复过于单一 | 体验像普通聊天机器人 | 增加玩家意图、关系状态和记忆标签约束 |
| 人设漂移 | NPC 说出不符合身份的话 | Reviewer 增加人设一致性检查 |
| 剧情泄露 | 提前透露后续章节信息 | Prompt 中区分可透露信息和禁止事项 |
| 布局遮挡 | 头像、状态栏或按钮影响体验 | 固定核心区域尺寸并做响应式检查 |
| 外部访问失败 | 用户打开链接后进入登录页或空白 | Demo 入口使用 `?demo=1` 并保留 fallback 提示 |
