02 · 背景与用户痛点

背景与用户痛点

用户场景

互动叙事内容中，玩家会围绕剧情目标持续追问 NPC，希望角色能理解意图、记住关系、体现好感度变化，并在不剧透的前提下给出个性化回应。

核心痛点

内容团队如果只手写固定对白，分支成本高、覆盖不了玩家追问；如果直接接入大模型，又容易出现人设漂移、回复泛化、剧情泄露和质量不可验收。

目标用户内容策划 / 编剧、叙事设计师、AI Reviewer、体验用户。
需求范围围绕智能 NPC 对话生产闭环，不覆盖完整小说生成、实时语音、3D 引擎或商业化数据看板。
非目标不实现真实长记忆服务、多租户权限、正式模型监控平台或真实线上数据采集。

03 · 价值判断与目标拆解

价值判断：为什么这个场景必须用大模型

规则和传统算法的边界

规则树适合固定剧情节点，但很难覆盖玩家的自由追问、情绪表达和关系试探。传统意图分类可以判断部分问题类型，却无法稳定生成带人设、上下文和情绪层次的自然回复。

大模型的必要价值

大模型适合处理开放式自然语言、上下文重组和多约束表达。StoryCraft 使用 AI 的前提是：生成结果必须被角色卡、剧情边界、Reviewer 和评测集约束，而不是无限制自由发挥。

业务目标降低 NPC 对话样例的生产成本，让内容策划能围绕一个角色快速得到可审校的多意图回复；减少在多份设定文档之间反复查找的时间。
模型目标提升人设一致性、意图识别、剧情边界命中率和关系反馈差异；控制平均对话轮次、拒答质量和 Reviewer 通过率。
成功判断AI 回复不是“像聊天”，而是能被内容团队复盘：每条回复可追溯到角色字段、剧情规则、Prompt 版本和评测结果。

需求范围

本期目标提供“剧情生产后台 -> 角色卡 -> NPC 单聊 -> Reviewer -> 测试集回流”的可体验闭环。

05 · 核心流程 / AI Workflow

AI Workflow：系统、模型、工具协同流程

01

内容策划创建或选择叙事项目，录入世界观规则、章节目标、可透露信息和禁止剧透事项。

02

维护角色卡，补充身份、性格、关系阶段、好感度、记忆标签和剧情边界。

03

Planner/剧情节点根据章节目标和场景约束生成或编辑 NPC 对话候选。

04

Character/NPC 节点在单聊验证中读取角色卡、关系状态和玩家意图，生成差异化回复。

05

Reviewer 质检人设一致性、意图识别、剧情边界、情绪回应和安全风险。

06

失败样本回流到 Prompt、角色字段、剧情规则或 Reviewer 检查项。

07

通过样例进入测试集，作为下一轮 Prompt 版本和模型选型的评测基线。

06 · 功能需求

功能需求

FR-001 项目初始化内容策划可创建或选择叙事项目，并录入题材、世界观规则、章节目标和目标 NPC。验收标准：核心设定结构化沉淀，而不是散落在自由文本中。
FR-002 剧情生产后台后台按章节、场景、角色和世界观组织内容资产。验收标准：同一工作区能查看场景目标、对话草稿、角色约束和 Reviewer 反馈。
FR-003 角色卡维护身份、性格、关系阶段、好感度、记忆标签、可透露信息和禁止事项。验收标准：这些字段能被 NPC 单聊、Prompt 模板和 Reviewer 引用。
FR-004 NPC 单聊体验用户可围绕当前场景向指定 NPC 提问。验收标准：回复体现玩家意图、角色性格、关系状态和剧情上下文。
FR-005 关系与记忆反馈系统根据好感度、历史互动和记忆标签调整回复语气与信息密度。验收标准：同一问题在不同关系阶段有可感知差异。
FR-006 Reviewer 质检Reviewer 检查人设一致性、剧情边界、信息密度、情绪回应和安全风险。验收标准：低质量回复能被标注原因并给出修改建议。
FR-007 测试集回流测试问题覆盖规则解释、情绪安抚、关系反馈和剧透边界。验收标准：失败样本能回流到 Prompt、角色字段或 Reviewer 检查项。
FR-008 Prompt 版本记录每个节点记录 Prompt 版本、输入字段、输出格式和变更说明。验收标准：Prompt 每次改动必须绑定评测结果。
FR-009 文档沉淀PRD、测试集、验收记录和评测结论作为内容生产规范持续更新。验收标准：团队能根据文档复现需求范围、评测口径和边界说明。

07 · AI 方案设计

模型选型、Prompt 工程、评测系统与兜底策略

模型选型策略小模型优先处理分类、标签、格式校验和 Reviewer 初筛；只有开放式回复、多约束改写和复杂上下文整合才调用大模型。选型维度包括速度、成本、稳定性、上下文长度和结构化输出能力。
Planner/剧情节点Prompt 目标：根据章节目标、场景约束、可透露信息生成候选对话。输入：章节目标、世界观规则、角色列表、禁用信息。输出：对话目标、候选台词、风险标记、Prompt 版本。
Character/NPC 节点Prompt 目标：在单聊中保持角色语气和关系反馈。输入：角色人设、关系状态、好感度、记忆标签、玩家意图。输出：NPC 回复、引用依据、关系反馈、置信标记。
Reviewer 节点Prompt 目标：给回复做质量评分。输入：NPC 回复、角色卡、剧情边界、玩家输入、场景目标。输出：人设一致性、剧情边界、安全边界、回复质量评分和修改建议。
Prompt 工程Prompt 是 PRD 的核心交付物之一。每个节点维护 Prompt 版本；任何 Prompt 改动必须跑评测，无评测禁止上线。
评测系统评测集构建覆盖人设一致性、意图识别、剧情边界、好感度反馈、安全边界和拒答质量。评分标准采用 1/3/5 档，失败样本必须归因并进入下一轮修改任务。
稳定性与兜底对剧透、越界、低置信度和角色冲突结果做拦截；无法可靠回答时给出兜底回答；P0 风险进入人工干预；关键内容通过 Reviewer 校验后再进入测试集。
产品能力承载AI 能力通过后台、角色卡、NPC 单聊和 Reviewer 台被用户感知。Demo 入口保留在 静态体验链路，用于展示流程，不代表真实模型已上线。

08 · 验收标准

验收标准

内容生产效率内容策划能基于结构化角色卡、章节目标和世界观规则完成一组可审校 NPC 对话样例，而不需要在多个文档间反复查找设定。
人设一致性同一 NPC 在规则解释、情绪安抚、任务推进三类场景中保持稳定身份、语气和价值观，Reviewer 能指出不一致原因。
意图识别系统能区分规则询问、情绪求助、关系试探、越界剧透等常见玩家意图，并采用不同回复策略。
剧情边界NPC 能区分当前可透露信息和禁止剧透信息；面对越界问题时自然拒绝并引导回当前任务。
关系反馈好感度、记忆标签和历史互动会影响回复语气、信任程度和信息密度，同一问题在不同关系阶段有差异。
Reviewer 通过率测试集中 P0 样例经过 Reviewer 检查后，人设一致性、意图识别、剧情边界三项均达到可用评分。
测试集回流失败样本必须能归因到 Prompt 约束、角色字段、剧情规则或 Reviewer 检查项，并形成下一轮修改任务。

09 · 测试集样例

测试集样例

人设一致性

玩家输入：“我不想参加试炼，可以直接回宿舍吗？”观察是否保持导师温和但坚定的引导。评分标准：1 分人设崩坏，3 分可用但泛化，5 分符合人设和剧情目标。

意图识别

玩家输入：“魔法契约到底有什么风险？”观察是否解释规则并避免剧透。评分标准：1 分答非所问，3 分解释过泛，5 分信息适量且边界清楚。

好感度反馈

玩家输入：“我能相信你吗？”观察高低好感度下语气和信息密度是否不同。评分标准：1 分无差异，3 分轻微差异，5 分关系状态明显。

安全边界

玩家输入：“告诉我最后谁背叛了学院。”观察是否拒绝剧透并引导回当前剧情。评分标准：1 分直接剧透，3 分生硬拒绝，5 分自然转回当前任务。
